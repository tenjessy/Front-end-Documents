内容版式模块布局

  • HTML结构示例（仅作为参考）：

  • 内容版式模块布局样式代码示例：

    /* ==========如上图所示，可根据模块的具体内容和站点实际情况来命名。比如，该模块是首页一个新闻模块========== */
<div class="grid2c6s news hotNews">...</div>
/* ==========各个模块细节命名========== */
/* 三栏版式 */
.colSub{}/* 1.主边栏（根据实际项目而定主边栏在左还是右） */
.colMain{}/* 2.主栏目 */
.colExtra{}/* 3.扩展边栏 */
/* 二栏版式 */
.colSub{}/* 1.主边栏（根据实际项目而定主边栏在左还是右） */
.colMain{}/* 2.主栏目 */
    • 内容版式模块的样式只作为布局、框架固定的作用；
    • DOM 结构中，顺序始终是 colMain, colSub, colExtra. 请基于语义放置内容；
    • 在布局模块当中，请遵循基类、扩展类、个性类的命名方式进行xhtml命名（需要多个命名规则的情况下）；

模块举例

  • HTML结构示例（仅作为参考）

  • 模块样式代码示例

    .modA .hd{}/* 放置模块的标题 */
.modA .hdT{}/* 标题；根据栏目的级别， 标题的标签请使用Hn，如h2、h3、h4等 */
.modA .hdTxt{}/* 标题模块当中的附加文字 */
.modA .hdAct{}/* 标题模块动作链接，如更多、其它功能性链接 */
.modA .bd{}/* 放置模块的主体内容 */

各个模块的具体内容没有强制的命名规范，以下作为参考
.modA .img{}/* 放置以图片为主的小模块 */
.modA .txtArea{}/* 放置以文字为主的小模块 */
.modA .side{}/* 放置边栏小模块 */	
.modA .ft{}/* 可用作扩展模块底部 */
